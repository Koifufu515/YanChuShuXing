# BankInsight 接口契约

> 契约版本：v1
> 冻结日期：2026-07-12
> 状态：目标契约已冻结，现有代码尚未全部实现。任何破坏性修改必须先更新本文并经团队确认。

## 一、设计原则

1. 外部 API 只暴露前端真正需要的字段，不暴露 Intent、Plan、Provider 等内部结构。
2. Pipeline 只编排组件，不依赖 FastAPI、Streamlit、SQLite 连接或具体模型 SDK。
3. LLM、规则生成器、数据库、安全检查器都可以按相同接口替换。
4. 组件用结构化结果或明确异常传递失败，禁止返回含义不清的 `None`。
5. 第一版同步执行即可；超时、后台任务和流式响应留到后续版本。
6. SQL 参数与 SQL 文本分离；DatabaseExecutor 不接受自然语言问题。

## 二、统一 JSON 类型

查询结果单元格只允许 JSON 可序列化标量：

```python
JsonScalar = str | int | float | bool | None
```

日期、时间和 Decimal 在 DatabaseExecutor 边界统一转换为 ISO 字符串或数值，前端不处理 Python 专有类型。

## 三、前端调用后端的查询 API

### 3.1 路由

```http
POST /api/v1/query
Content-Type: application/json
```

现有 `/api/v1/ask` 在下一阶段迁移为 `/api/v1/query`；迁移期间可保留兼容别名，但文档和前端只使用新路径。

### 3.2 请求结构 `QueryRequest`

```json
{
  "question": "查询客户王总的账户余额",
  "user_id": "demo_user",
  "conversation_id": "demo_session"
}
```

| 字段 | 类型 | 必填 | 约束 | 说明 |
|---|---|---:|---|---|
| `question` | string | 是 | 去空格后2—500字符 | 用户原始问题 |
| `user_id` | string | 是 | 1—64字符，仅演示身份 | 后续权限和审计接口的稳定入口 |
| `conversation_id` | string/null | 否 | 最长128字符 | 多轮会话标识；首版可不使用 |

前端不得传入 SQL、数据库路径、角色权限或表名白名单。

### 3.3 成功响应 `QueryResponse`

```json
{
  "request_id": "01J2BANKINSIGHT001",
  "question": "查询客户王总的账户余额",
  "sql": "SELECT customer_name, current_balance FROM ... WHERE customer_name = :customer_name",
  "columns": ["客户姓名", "账户余额"],
  "rows": [["王总", 5200000]],
  "summary": "王总当前账户余额为520万元。",
  "warnings": [],
  "error": null
}
```

| 字段 | 类型 | 必有 | 说明 |
|---|---|---:|---|
| `request_id` | string | 是 | 后端生成，贯穿日志与错误定位 |
| `question` | string | 是 | 原始问题原样返回 |
| `sql` | string/null | 是 | 成功时为实际执行 SQL；生成前失败时为 null |
| `columns` | string[] | 是 | 列名，顺序与每行单元格一致 |
| `rows` | JsonScalar[][] | 是 | 二维数组；失败时为空数组 |
| `summary` | string/null | 是 | 业务摘要；无结果或失败可为 null |
| `warnings` | string[] | 是 | 非致命提示，如结果被截断 |
| `error` | ErrorDetail/null | 是 | 成功时必须为 null |

约束：每一行的长度必须等于 `columns` 长度；默认最多返回1000行；若截断必须在 `warnings` 中说明。

### 3.4 统一错误对象 `ErrorDetail`

```json
{
  "code": "UNSUPPORTED_QUESTION",
  "message": "首版暂不支持该问题，请尝试预置问题。",
  "retryable": false
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | string | 稳定、可判断的机器错误码 |
| `message` | string | 面向用户的中文说明，不包含堆栈和密钥 |
| `retryable` | boolean | 前端是否可以提示用户重试 |

失败响应仍使用完整 `QueryResponse` 外壳：`columns=[]`、`rows=[]`、`summary=null`，并填写 `error`。

### 3.5 HTTP 状态与错误码

| HTTP | 错误码示例 | 场景 |
|---:|---|---|
| 200 | - | 查询成功，包括结果为空 |
| 400 | `INVALID_QUESTION`、`UNSUPPORTED_QUESTION` | 请求格式合法但业务无法处理 |
| 403 | `SQL_REJECTED`、`ACCESS_DENIED` | SQL 安全或权限拒绝 |
| 422 | `REQUEST_VALIDATION_ERROR` | 字段缺失、长度或类型错误 |
| 500 | `INTERNAL_ERROR` | 未预期服务端错误 |
| 502 | `LLM_PROVIDER_ERROR` | 模型服务返回异常结果 |
| 503 | `DATABASE_UNAVAILABLE`、`LLM_UNAVAILABLE` | 依赖暂时不可用 |
| 504 | `QUERY_TIMEOUT`、`LLM_TIMEOUT` | 查询或模型超时 |

服务端日志保存原始异常；API 响应不得返回 Python 堆栈。

## 四、Query Pipeline 接口

```python
class QueryPipeline(Protocol):
    def run(self, request: QueryRequest) -> QueryResponse:
        ...
```

### 输入

- 只接收领域 `QueryRequest`，不得接收 FastAPI `Request`。
- `user_id` 用于构造最小 `UserContext`，但第一版不实现完整权限。

### 输出

- 所有已知失败都归一化为 `QueryResponse.error`。
- 只有编程错误可以向上抛出，并由 API 全局异常处理器转为 `INTERNAL_ERROR`。

### 第一版固定编排顺序

```text
QueryRequest
  -> Schema/Metric Context Builder
  -> LLMProvider 或 RuleProvider
  -> SQLSafetyChecker
  -> DatabaseExecutor
  -> ResultFormatter
  -> QueryResponse
```

Pipeline 不得：

- 直接调用某个模型厂商 SDK；
- 自己创建 SQLite 连接；
- 在内部拼接网页 HTML；
- 复制安全检查逻辑；
- 读取 FastAPI 全局对象。

## 五、LLM Provider 接口

为兼容首版规则回退，接口命名使用 `SQLProvider` 更准确；真实大模型实现仍命名为 `LLMProvider`。

```python
class SQLProvider(Protocol):
    def generate_sql(
        self,
        question: str,
        schema_context: str,
        metric_context: str,
    ) -> str:
        ...
```

### 输入

| 参数 | 说明 |
|---|---|
| `question` | 已去除首尾空格的用户问题 |
| `schema_context` | Resolver 生成的允许表字段和连接关系文本 |
| `metric_context` | Resolver 生成的指标口径文本 |

### 输出

- 只返回一条 SQL 字符串；不得包含 Markdown 代码围栏或解释文字。
- 不得执行 SQL，也不得自行判断用户权限。
- SQL 必须使用项目配置的方言，首版为 SQLite。

### 异常

| 异常 | 触发场景 | Pipeline 映射 |
|---|---|---|
| `UnsupportedQuestionError` | 规则 Provider 不支持问题 | HTTP 400 / `UNSUPPORTED_QUESTION` |
| `ProviderUnavailableError` | 模型服务不可用 | HTTP 503 / `LLM_UNAVAILABLE` |
| `ProviderTimeoutError` | 模型请求超时 | HTTP 504 / `LLM_TIMEOUT` |
| `InvalidProviderOutputError` | 输出为空或不是单条 SQL | HTTP 502 / `LLM_PROVIDER_ERROR` |

第一版 `RuleProvider` 和后续 `LLMProvider` 必须实现同一接口。

## 六、Database Executor 接口

```python
class DatabaseExecutor(Protocol):
    def execute_query(
        self,
        sql: str,
        parameters: dict[str, JsonScalar] | None = None,
        max_rows: int = 1000,
    ) -> QueryResult:
        ...
```

### 统一结果 `QueryResult`

```python
@dataclass(frozen=True)
class QueryResult:
    columns: list[str]
    rows: list[list[JsonScalar]]
    row_count: int
    truncated: bool
    duration_ms: float
```

约束：

- 只使用只读 SQLite 连接；安全检查通过后才调用。
- 所有动态值必须参数绑定，禁止由 Executor 拼接用户输入。
- `max_rows` 有效范围1—1000；Executor 最多读取 `max_rows + 1` 行判断截断。
- 每次调用独立获取并关闭连接；不得把 Cursor 返回给上层。
- 不把 SQLite 异常原文直接暴露给前端。

异常映射：

| 异常 | Pipeline 错误码 |
|---|---|
| `DatabaseUnavailableError` | `DATABASE_UNAVAILABLE` |
| `QueryExecutionError` | `QUERY_EXECUTION_ERROR` |
| `QueryTimeoutError` | `QUERY_TIMEOUT` |

## 七、SQL Safety Checker 接口

```python
class SQLSafetyChecker(Protocol):
    def validate(
        self,
        sql: str,
        user_context: UserContext | None = None,
    ) -> SafetyResult:
        ...
```

```python
@dataclass(frozen=True)
class UserContext:
    user_id: str
    allowed_tables: frozenset[str] = frozenset()
    denied_columns: frozenset[str] = frozenset()


@dataclass(frozen=True)
class SafetyResult:
    allowed: bool
    warnings: list[str]
    error_code: str | None
    error_message: str | None
    referenced_tables: list[str]
```

约束：

- 任何 SQL 都必须先验证再执行。
- 第一版至少检查：单语句、只读 SELECT/CTE、系统表、允许表、拒绝字段。
- `allowed=False` 时 Pipeline 立即返回 HTTP 403 / `SQL_REJECTED`，不得调用 Executor。
- Checker 不修改 SQL；后续如需自动加 LIMIT，应由独立 Query Policy 组件处理。

现有 `SqlSafetyValidator` 可通过适配器满足该接口，不要求本阶段重写。

## 八、Result Formatter 接口

虽然原任务未单独要求，但为了保持 Pipeline 简单，固定最小接口：

```python
class ResultFormatter(Protocol):
    def format_summary(
        self,
        question: str,
        result: QueryResult,
    ) -> str | None:
        ...
```

第一版可用确定性模板生成摘要，不依赖 LLM。Formatter 不访问数据库，也不改变结果行。

## 九、审计接口预留

```python
class AuditLogger(Protocol):
    def record(self, event: AuditEvent) -> None:
        ...
```

第一版使用 `NoOpAuditLogger`，只固定调用位置：请求开始、SQL 被拒绝、查询成功、查询失败。暂不实现数据库审计表和完整权限系统。

## 十、前后端协作约定

前端成员只依赖以下内容：

- `POST /api/v1/query`；
- `QueryRequest` 和 `QueryResponse` JSON；
- HTTP 状态码与错误码表。

前端不得导入后端 Python 文件，也不得直接打开 SQLite。后端可以调整内部 Intent、Plan、Schema 召回逻辑，只要外部 v1 契约不变。

## 十一、契约示例

### 空结果

```json
{
  "request_id": "req_empty_001",
  "question": "查询不存在客户的账户余额",
  "sql": "SELECT ...",
  "columns": ["客户姓名", "账户余额"],
  "rows": [],
  "summary": "未查询到符合条件的数据。",
  "warnings": [],
  "error": null
}
```

### 不支持的问题

```json
{
  "request_id": "req_unsupported_001",
  "question": "预测明年的股票价格",
  "sql": null,
  "columns": [],
  "rows": [],
  "summary": null,
  "warnings": [],
  "error": {
    "code": "UNSUPPORTED_QUESTION",
    "message": "首版暂不支持该问题，请尝试预置问题。",
    "retryable": false
  }
}
```

### SQL 被安全策略拒绝

```json
{
  "request_id": "req_rejected_001",
  "question": "删除全部客户数据",
  "sql": "DELETE FROM customer_info",
  "columns": [],
  "rows": [],
  "summary": null,
  "warnings": [],
  "error": {
    "code": "SQL_REJECTED",
    "message": "生成的查询未通过只读安全检查。",
    "retryable": false
  }
}
```

## 十二、版本变更规则

- 新增可选响应字段属于兼容变更，可继续使用 v1。
- 删除字段、改变类型、改变字段含义或错误码语义属于破坏性变更，必须升级到 v2。
- 组件内部模型不属于外部 API 契约，但 Protocol 方法签名变更必须通知对应模块负责人。
- 下一阶段实现应先写契约测试，再编写 Provider、Executor 和 Pipeline。
