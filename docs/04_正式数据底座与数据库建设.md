# 04号负责人：正式数据底座与数据库建设

> 对应 GitHub Issue：#4“五张官方表的数据审计、规范化入库与查询底座建设”
> 当前状态：任务规范已就绪，等待数据负责人开展完整审计与建库。

## 一、任务目标

04号负责人将五张官方表整理为结构清晰、数据准确、能够被后端稳定查询的正式数据底座。工作范围包括机构信息表、指标清单表、衍生维度说明、指标数据表和问题答案清单。

五张表根据用途分别进入业务数据库、业务配置或评测资产：

- 机构信息表、指标清单表和指标数据表构成主要查询底座；
- 衍生维度说明支持业务规则整理和后端语义配置；
- 问题答案清单作为独立评测资产提供给06号使用。

正式数据环境与Demo技术基线并行保留。Demo用于开发、公开演示和架构回归，Real环境用于官方数据验证和比赛评测。

## 二、五张表的数据审计

04号逐表检查字段名称、数据类型、主键候选、唯一性、缺失值、重复值、日期格式和表间关系，重点完成以下核对：

1. 机构编号在机构信息表中保持唯一；
2. 指标编号在指标清单表中保持唯一；
3. 指标数据表中的机构编号均可关联到机构信息表；
4. 指标数据表中的指标编号均可关联到指标清单表；
5. 同一机构、同一指标和同一日期对应唯一指标记录；
6. 日期、数值、单位和精度采用统一格式；
7. 导入前后的记录数量保持一致；
8. 异常记录具有问题位置、问题类型、影响范围和处理结果。

数据审计报告按工作表汇总检查范围、规则、问题数量、处理状态和证据。完整数据值保存在受控目录，公开报告使用字段模板、脱敏样例和聚合统计。

## 三、正式数据库结构设计

正式查询底座至少包含机构表、指标表和指标事实表。

### 机构表

机构表保存机构编号、机构名称及官方数据中存在的机构属性。机构编号作为稳定主键，机构名称和属性用于查询展示与筛选。

### 指标表

指标表保存指标编号、标准名称、指标说明、单位和数值精度。指标编号作为稳定主键，指标定义以官方指标清单为权威来源。

### 指标事实表

指标事实表保存机构编号、指标编号、数据日期和指标值。机构编号和指标编号分别关联机构表与指标表，并使用“机构编号、指标编号、数据日期”的组合唯一约束。

数据库结构需要支持以下官方题库场景：

- 单机构、单日期、单指标查询；
- 同一机构的跨期比较；
- 同一指标的机构排名；
- 指定机构和指标的时间趋势；
- 多机构或多指标对比；
- 基于基础指标的差值、平均值、比例和排名计算。

衍生维度说明根据后端使用方式整理为元数据表或配置文件，并与05号确认后的业务口径保持一致。问题答案清单进入独立评测存储，与正式业务查询表分开管理。

## 四、可重复的数据导入流程

04号实现可重复执行的数据库初始化和数据导入脚本，流程包括：

1. 从项目约定的受控位置读取官方工作簿；
2. 校验工作表名称和必要字段；
3. 统一日期、数值和文本字段格式；
4. 创建正式数据库结构、约束和索引；
5. 使用事务完成批量导入；
6. 输出每张表的读取数量、成功数量和异常数量；
7. 完成导入前后的记录数量对账；
8. 在导入失败时记录明确错误并回滚事务。

同一份官方工作簿在不同成员电脑上运行该流程，应生成结构、记录数量和关键抽样值一致的正式数据库。初始化脚本提供明确参数，用于选择Demo或Real环境及输入文件位置。

## 五、查询支持与数据库验证

04号根据高频筛选和关联条件设计索引，并提供典型查询验证脚本。至少验证：

1. 查询某机构某日期的单项指标；
2. 比较同一机构两个日期的指标值；
3. 查询某指标在全部机构中的排名；
4. 查询某机构某指标的一段时间趋势；
5. 查询多个机构或多个指标的对比结果；
6. 计算基础指标支持的差值、平均值、比例和排名结果。

每个验证场景记录输入条件、涉及表字段、返回结构、执行状态和耗时。必要索引根据典型查询的筛选、关联和排序字段确定。

数据库Schema说明写明表名、字段名、字段类型、主外键关系、唯一约束、单位、精度、索引和典型查询方式，作为后端Schema上下文与SQL生成链路的正式依据。

## 六、协作流程

### 与05号业务语义负责人

04号提供正式指标编号、机构编号、字段名称、单位、精度和数据库映射。05号据此审核题库表达、衍生指标和复合业务计算口径。双方共同确认衍生维度采用元数据表还是配置文件表达。

### 与06号测试负责人

06号在批量评测中记录数据库执行错误、字段关联问题、缺失记录和异常结果，并向04号提交问题编号、实际SQL、数据库错误或结果证据。04号完成数据检查和修正后，由06号重新运行原问题和受影响回归集。

### 与项目负责人

项目负责人审核数据库结构、公共配置、初始化方式和后端接入方案，并负责将正式数据库接入统一查询链路。涉及Schema、公共配置和数据库Adapter的修改通过独立Pull Request完成。
