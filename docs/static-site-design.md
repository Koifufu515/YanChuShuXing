# GitHub Pages 静态演示站设计

## 目标

构建可由 GitHub Pages 直接托管的“言出数行”介绍页和静态问数工作台。站点用于公开展示项目思路、数据治理边界和交互流程，不连接 FastAPI、SQLite、真实工作簿或题库答案。

## 目录与部署

静态资源放入 `site/`：`index.html`、`assets/styles.css`、`assets/app.js`、`assets/demo-data.js`。GitHub Actions 在推送 `main` 后将 `site/` 发布至 GitHub Pages。站点不需要 Node、构建步骤、API Key 或运行时服务器。

## 页面

首页展示产品名称、银行智能问数闭环、系统架构、数据治理和团队协作边界。首屏提供进入工作台入口。

工作台提供六个业务场景、文本问题输入、预置示例问题和三种预置模拟结果。匹配问题后，页面依次显示查询状态、业务结论、指标卡、表格或趋势图，以及折叠的模拟技术信息。

当输入不匹配预置样例时，显示条件不足状态；当用户选择特定演示问题时，显示无数据或查询失败状态。所有状态均在浏览器本地模拟。

## 数据边界

`assets/demo-data.js` 只包含虚构机构、虚构问题、虚构指标值和虚构结果，不得复制官方题目、标准答案、真实机构名称、真实指标值、真实 SQL 或 private 目录内容。页面必须明确标注“静态模拟演示”。

## 验收

- 直接打开 `site/index.html` 可操作；
- 不请求本地或远程后端；
- 无真实数据、Token、`.env` 或 private 路径进入静态资源；
- GitHub Pages 工作流仅发布 `site/`；
- 桌面与移动宽度下文字与控件不重叠。
