# 竞赛数据与智算平台资料使用说明

## 资料状态

团队已经通过大赛官方智算平台取得本赛题数据包和更新版平台操作手册。初步安全审计确认，数据包包含业务指标与受控评测答案，平台手册包含注册验证流程和个人联系方式。两份原件均属于参赛过程中的受控资料，不进入当前 Public GitHub。

经只读结构核对，数据覆盖13家机构、21项经营指标、10项衍生维度说明、486个数据日期、132,678条指标事实记录和200道官方问题。官方问题包含训练、验证和测试分区以及简单、普通、复杂三个难度层级。这些汇总信息可以用于公开说明项目规模，但题目原文、标准答案、逐题结果和真实数据内容仍按受控资产管理。

公开仓库只保存本文、脱敏后的数据治理规则、无数据 ETL 代码、映射模板和确定性 Demo 数据。原始压缩包、工作簿、问题答案、完整字段映射、真实查询结果和平台手册原件保存在被 Git 忽略的本地目录或项目负责人批准的私有共享位置。

## 双环境原则

**Demo 环境**用于日常开发、自动化测试、公开演示和 GitHub 协作，继续使用 `data/processed/bankinsight.db`。该数据库由初始化脚本确定性生成，不含真实客户信息。

**Real 环境**用于数据审计、字段映射、清洗脱敏、Gold SQL 对账和受控业务验证。原始文件与处理结果存放在 `data/private/`、`data/real/` 或 `archive_local/`，这些路径均不得被 Git 跟踪。Real 环境尚未接入当前公开 Demo，后续实现必须通过独立 Issue、测试和 Pull Request。

两套环境不得覆盖或共用数据库文件。ETL 需要参数化输入路径，执行前校验文件版本，执行后完成行数、类型、主键、关系和异常对账。测试报告必须标明使用 Demo 还是 Real，公开报告不得包含真实字段样例、标准答案或敏感结果。

后续 `evaluation/` 目录只保存评测框架、加载接口、指标计算和聚合报告模板；官方200题及答案从被Git忽略的私有位置加载。每次修改Rule、LLM、Prompt、Schema或数据库后，应在受控环境重复评测，并输出Rule命中率、LLM成功率、SQL生成与执行成功率、答案正确率、耗时和分难度表现。

## 智算平台操作摘要

官方平台流程包括账号注册与登录、赛题数据下载、可选算力资源申请和备测成果提交。注册及下载需要使用申请数据时登记的信息完成验证；数据可通过浏览器、SFTP 或平台提供的脚本获取。算力资源可按需要申请 GPU、CPU、内存、硬盘和使用时间，审批后通过平台进入计算环境。

备测成果需要提供可运行的完整源代码、必要的本地模型文件、依赖和运行步骤清楚的部署文档，以及项目压缩包；公网地址、端口和访问路径按部署情况填写。团队提交前应在接近标准环境的干净环境完成安装、数据库初始化、后端、网页和核心查询验证。

原手册中的平台地址、手机号、邮箱、验证码流程截图和支持人员联系方式不在公开仓库复述。队员需要原件时，应从团队受控目录获取，并按官方平台当前说明操作。

## Git 提交检查

每次提交前检查：`git status --ignored` 能看到真实数据目录处于忽略状态；`git ls-files` 不包含赛题压缩包、工作簿、答案、手册原件和真实结果；Issue、PR、测试日志和截图不出现原始数据内容、个人信息、平台凭据或支持人员联系方式。若误提交，应立即停止推送、通知项目负责人并按安全流程清理 Git 历史。
