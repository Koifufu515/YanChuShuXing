# 数据库与指标字典

## 建模原则

- 客户、分行、客户经理和产品是维度实体；交易、理财申购、渠道行为和风险事件是事实事件；账户和贷款合同兼具实体与余额事实属性。
- 所有金额字段以人民币元存储，展示层再转换为万元或亿元，避免 SQL 中单位混乱。
- 时间字段使用 ISO 日期/时间格式；“期末余额”首版取当前快照，后续增加账户和贷款月末快照表以支持严格历史余额。
- 事实表保留业务发生时的 `branch_id`，避免客户归属变更影响历史统计。
- 客户、账户、交易、贷款和风险标识均属于高敏感字段，默认只允许聚合查询，明细展示需脱敏和授权。

## 十张表字段说明

### branch_info：分行信息表

| 字段 | 类型 | 键 | 含义 |
|---|---|---|---|
| branch_id | TEXT | PK | 分行唯一编号 |
| branch_name | TEXT |  | 分行名称 |
| branch_level | TEXT |  | 总行/一级/二级分行 |
| parent_branch_id | TEXT | FK | 上级分行 |
| region_name | TEXT |  | 华东、华南等经营区域 |
| province_name / city_name | TEXT |  | 省份和城市 |
| open_date | DATE |  | 开业日期 |
| status | TEXT |  | 机构状态 |

支持地区分布、分行层级、经营排名和跨区域比较。

### customer_manager：客户经理表

| 字段 | 类型 | 键 | 含义 |
|---|---|---|---|
| manager_id | TEXT | PK | 客户经理编号 |
| manager_name | TEXT |  | 姓名，展示需脱敏 |
| branch_id | TEXT | FK | 所属分行 |
| hire_date | DATE |  | 入职日期 |
| manager_level | TEXT |  | 初/中/高级 |
| status | TEXT |  | 在职状态 |

支持经理管户规模、客户 AUM、理财销量和风险客户分布。

### customer_info：客户基础信息表

| 字段 | 类型 | 键 | 含义 |
|---|---|---|---|
| customer_id | TEXT | PK | 客户编号，默认哈希展示 |
| gender / birth_year | TEXT / INTEGER |  | 基础人口属性 |
| province_name / city_tier | TEXT |  | 客户所在地区与城市等级 |
| occupation_type | TEXT |  | 职业类型 |
| customer_level | TEXT |  | 大众、富裕、高净值 |
| risk_preference | TEXT |  | C1-C5 风险承受等级 |
| register_date | DATE |  | 开户日期 |
| branch_id / manager_id | TEXT | FK | 归属分行和客户经理 |
| customer_status | TEXT |  | 活跃、休眠或销户 |

支持新增客户、客户分层、画像、流失观察和客群风险分析。

### account_info：账户信息表

| 字段 | 类型 | 键 | 含义 |
|---|---|---|---|
| account_id | TEXT | PK | 账户编号 |
| customer_id / branch_id | TEXT | FK | 客户和开户分行 |
| account_type | TEXT |  | 活期、定期、理财现金账户 |
| currency_code | TEXT |  | 币种，首版以 CNY 为主 |
| open_date / close_date | DATE |  | 开销户日期 |

### transaction_detail：交易流水表

| 字段 | 类型 | 键 | 含义 |
|---|---|---|---|

## 七大主题指标口径

| 主题 | 指标 | 计算口径 | 主要依赖 |
|---|---|---|---|
| 客户 | 有效客户数 | 期末 `customer_status='ACTIVE'` 的去重客户数 | customer_info |
| 客户 | 新增客户数 | 统计期内 `register_date` 落入区间的去重客户数 | customer_info |
| 客户 | 客户 AUM | 有效账户余额 + 理财持有金额；贷款不抵减，首版称“管理资产代理指标” | account_info, product_purchase |
| 客户 | 客户流失率 | 期初活跃客户中，观察期内无有效行为且转为休眠/销户的客户占比 | customer_info, channel_behavior, transaction_detail |
| 存款 | 存款余额 | 有效人民币活期与定期账户期末余额之和 | account_info |
| 存款 | 活期占比 | 活期存款余额 / 活期与定期存款余额 | account_info |
| 存款 | 存款集中度 | Top 10 客户存款余额 / 全部分组存款余额 | account_info |
| 贷款 | 贷款余额 | 有效和违约未结清合同的 `outstanding_balance` 之和 | loan_info |
| 贷款 | 新增贷款金额 | 统计期内发放合同的 `original_amount` 之和 | loan_info |
| 贷款 | 贷款逾期率 | `SUM(overdue_amount) / SUM(outstanding_balance)` | loan_info |
| 贷款 | 不良贷款率 | 次级、可疑、损失类余额 / 全部未结清贷款余额 | loan_info |
| 理财 | 理财申购金额 | 统计期申购记录的 `purchase_amount` 之和 | product_purchase |
| 理财 | 理财购买客户数 | 统计期内申购理财的去重客户数 | product_purchase |
| 理财 | 复购率 | 历史至少购买过一次且本期再次购买的客户 / 本期购买客户 | product_purchase |
| 分行 | 分行客户规模 | 按归属分行统计有效客户数 | branch_info, customer_info |
| 分行 | 综合经营得分 | 对客户增量、存款增量、贷款质量、理财销量标准化后按配置权重加权 | 多主题表 |
| 渠道 | MAU | 自然月内指定渠道至少一次成功有效行为的去重客户数 | channel_behavior |
| 渠道 | 交易转化率 | 成功交易客户数 / 渠道活跃客户数 | channel_behavior |
| 渠道 | 交易失败率 | 失败交易尝试次数 / 全部交易尝试次数 | channel_behavior |
| 风险 | 高风险事件数 | 高风险且非误报的去重事件数 | risk_event |
| 风险 | 风险事件关闭率 | 已关闭事件 / 非误报事件 | risk_event |

注意：仅使用 `account_info.current_balance` 无法严谨回溯历史月末存款余额。正式第二版应增加 `account_balance_snapshot` 和 `loan_balance_snapshot` 月末快照表；首版演示趋势可用交易净流入重建或生成快照视图，但必须在口径卡中说明。
