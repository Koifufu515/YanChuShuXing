# 数据库与指标字典

> 数据模型版本：0.1.0
> 对应产品版本：0.5.2
> 校准日期：2026-07-14
> 状态：当前正式文档，以 `sql/schema.sql`、YAML 配置和 SQLite 实库为准

## 当前实现边界

- DDL、`config/schema.yml` 与演示数据库均包含同一组10张表，字段集合一致。
- `config/metrics.yml` 当前登记19项指标口径；“已登记”表示语义层可提供定义，不等于已有完整演示数据或已通过 Gold 问题验证。
- 当前确定性演示数据包含3名虚构客户、3家分行、1名客户经理、4个账户和4笔交易。贷款、理财、渠道行为与风险事件表结构已建立，但当前没有数据。
- 当前端到端稳定验证仅覆盖有效客户数、客户账户余额和客户月度交易汇总。其余指标属于已登记但待补数据与查询验证的能力。
- 所有编号和属性均为模拟数据，不含姓名、手机号、证件号或真实银行客户信息。

## 建模原则

- 客户、分行、客户经理和产品是维度实体；交易、理财申购、渠道行为和风险事件是事实事件；账户和贷款合同兼具实体与余额事实属性。
- 所有金额字段以人民币元存储，展示层再转换为万元或亿元，避免 SQL 中单位混乱。
- 时间字段使用 ISO 日期/时间格式；“期末余额”首版取当前快照，后续增加账户和贷款月末快照表以支持严格历史余额。
- 事实表保留业务发生时的 `branch_id`，避免客户归属变更影响历史统计。
- Schema 元数据使用 `public`、`internal`、`confidential`、`restricted` 四级敏感标签。当前 Demo Safety 只显式禁止 `manager_name`，尚未实现完整行列级权限；敏感标签是治理元数据，不能描述成已经落地的生产权限体系。

## 敏感等级摘要

| 表 | public | internal | confidential | restricted |
|---|---|---|---|---|
| branch_info | region/province/city | 机构编号、名称、层级、日期、状态 | - | - |
| customer_manager | - | branch_id、级别、状态 | manager_id、姓名、入职日期 | - |
| customer_info | - | city_tier、branch_id | 性别、出生年、省份、职业、客户等级、开户日期、manager_id、状态 | customer_id、风险偏好 |
| account_info | - | branch_id、币种 | 账户类型、开销户日期 | account_id、customer_id、余额、账户状态 |
| transaction_detail | - | branch_id | 交易类型、方向、渠道、交易状态 | 交易/账户/客户编号、时间、金额、异常标记 |
| loan_info | - | branch_id | manager_id、贷款类型、发放/到期日 | 贷款/客户编号及全部金额、利率、逾期、五级分类和状态 |
| wealth_product | - | 全部字段 | - | - |
| product_purchase | - | product_id、branch_id | manager_id、购买渠道 | 申购/客户编号、日期、金额、到期日和状态 |
| channel_behavior | - | branch_id | 渠道、事件、设备、状态 | 行为/客户/会话编号与事件时间 |
| risk_event | - | branch_id | - | 除 branch_id 外全部字段 |

逐字段敏感等级以 `config/schema.yml` 为唯一机器可读来源。

## 当前三条 Gold 问题

| 自然语言问题 | 主要指标/口径 | 涉及表 | 预期结果 |
|---|---|---|---|
| 查询有效客户数量 | `active_customer_count` | customer_info | 2户 |
| 查询客户C001的账户余额 | 有效账户余额汇总 | customer_info, account_info | 600.00万元 |
| 查询客户C001在2026年6月的交易汇总 | 交易四指标 | transaction_detail | 3笔，流入10万元、流出5万元、净流入5万元 |

“Gold”表示规则 SQL、Safety、只读执行、模板解释和 API 均有自动化测试，不表示系统只允许这三种问法。Hybrid 的非规则问题可进入 LLM 扩展，但成功范围受当前 Schema、指标语义和模型输出约束。

## 尚未进入指标配置的规划项

以下指标只保留为后续产品规划，不属于当前已实现能力：客户 AUM、客户流失率、存款集中度、理财复购率、分行综合经营得分、渠道交易失败率。

如要实现这些指标，必须先补充唯一指标 ID、公式、粒度、默认过滤、依赖字段、演示数据、Gold SQL 和测试，再更新本文。

## 数据时间口径限制

`account_info.current_balance` 和 `loan_info.outstanding_balance` 是当前快照，无法严谨回溯历史月末余额。后续如实现真实趋势分析，应增加账户和贷款月末快照表；不能把交易净流简单等同于完整余额变化。
