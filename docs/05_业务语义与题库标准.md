# 05号负责人：业务语义与题库标准

> 对应 GitHub Issue：#5“官方题库语义映射、衍生口径整理与题库异常审核”
> 资料边界：公开仓库保存规则、模板和最小核查示例；完整题目、官方答案和逐题审核结果保存在 `data/private/business_semantics/`。

## 一、角色定位

05号负责人负责官方题库语义映射、衍生指标与复合分析口径整理以及题库异常审核，并将确认结果整理为后端与评测可以引用的标准业务资产。技术成员根据确认后的业务资产完成系统配置和查询链路实现；06号执行批量评测；项目负责人确定异常题处理策略与Rule/LLM路由方案。

05号的核心价值不是“多写一份说明”，而是把题库语言与官方业务定义之间的关系审核清楚，使技术成员能够据此配置语义层、实现查询，并使测试负责人能够区分技术错误、口径理解错误和官方题目异常。

## 二、工作方式

### Codex 生成候选初稿

Codex读取五张官方表，自动提取：

- 题库中出现的指标别称和省略表达；
- 时间范围、比较基准和排名表达；
- 需要多个基础指标计算的衍生指标；
- 同时涉及多个指标的复合分析概念；
- 题目、官方答案、指标事实和衍生规则之间的疑似冲突。

候选内容保留来源问题编号和官方定义引用，并统一标记为“待审核”。经过05号确认后，技术成员再将对应内容转换为正式业务配置。

### 05号逐项审核

05号按顺序对照《机构信息表》《指标清单表》《衍生维度说明》《指标数据表》和《问题答案清单》，确认：

1. 表达是否真的出现在官方题库；
2. 对应的官方指标或衍生口径是否准确；
3. 公式、方向、单位、时间基准和输出形式是否明确；
4. 疑似异常是否有数据证据；
5. 审核状态是待审核、已确认、需讨论还是已驳回。

Codex负责生成候选初稿和提取结果，05号负责逐项审核与确认；技术成员负责把确认后的资产转换为系统配置，06号负责使用异常清单和标准口径开展评测。

## 三、题库表达映射

题库不一定完整使用《指标清单表》和《衍生维度说明》中的标准名称。05号需要记录“题库实际表达—官方指标编号或衍生口径—来源问题编号”的关系。

该资产定位为官方题库表达映射表，收录范围为题库中真实出现且能够追溯到具体问题编号的表达。

### 指标表达示例

`TRAIN-S-07` 使用“存款”这一省略表达，实际查询的是官方指标 `ZB001 各项存款余额`。该映射帮助语义层把题库中的短表达连接到权威指标定义，收录内容以题库中实际出现的表达为准。

### 时间与比较口径示例

`TRAIN-M-01` 使用“和2024年末相比变化了多少”，对应《衍生维度说明》中的“较年初”，其官方定义为“当日值减2024年12月31日值”。审核时同时记录比较基准、运算方向和结果单位，以完整表达该时间口径。

### 排序表达示例

`TRAIN-S-07` 中“三家谁存款最多”表示限定三个机构后，按 `ZB001` 降序取第一。排序表达应记录候选机构范围、排序方向和返回数量，避免把“最多”错误理解成全省排名。

公开模板见 [`business_semantics/题库表达映射表.csv`](business_semantics/题库表达映射表.csv)。

## 四、衍生指标与复合分析口径

此类内容不是简单同义词映射。05号必须明确涉及哪些基础指标、采用什么公式、时间和机构范围是什么、单位如何转换，以及结果应返回单值、列表、排名还是综合分析。

### 新增计算指标示例：存贷比

`TRAIN-M-25` 查询“存贷比”。它不是21项基础指标中的独立字段，应由两个官方指标计算：

```text
存贷比 = ZB002 各项贷款余额 / ZB001 各项存款余额 × 100%
```

该题是指定机构、指定日期的单值查询。审核时需要确认两个基础指标采用同一机构和日期，分母不为零，输入单位一致，输出为百分比，并记录舍入规则由评测规范统一处理。

### 复合业务概念示例：风险指标

`VAL-H-07` 使用“风险指标”这一复合概念，题目要求同时分析：

- `ZB013 不良贷款率`；
- `ZB015 拨备覆盖率`；
- `ZB017 逾期贷款率`；
- `ZB016 资本充足率`。

“风险指标”在该题中对应四项指标组成的结构化列表，并按题目要求补充排名或全省比较。05号负责固定概念包含关系和业务含义，技术成员据此实现SQL连接、聚合和路由。

公开模板见 [`business_semantics/衍生指标与复合分析口径表.csv`](business_semantics/衍生指标与复合分析口径表.csv)。

## 五、题库异常审核

05号记录问题编号、原问题、官方答案、冲突点、影响范围和建议处理方式，形成题库异常清单。项目负责人根据清单决定继续按官方答案判分、标记为异常题，或从某项统计中单独排除。

完整原文、答案和核查数值只保存在私有异常清单。公开仓库仅保留问题编号、异常类型和脱敏后的冲突说明。

### 方向表述冲突：TRAIN-M-46

受控核查显示，题目使用的比较方向与指标事实及答案结果方向不一致，可能影响答案规范化和业务解释。具体题目、答案和数值证据保存在私有异常清单。

### 极值判断错误：TRAIN-H-14

受控核查显示，题目对应答案中的最高值与最低值方向疑似颠倒。该题记录为“极值判断疑似错误”，具体数值证据保存在私有异常清单，等待项目负责人确定评测策略。

### 排名口径冲突：TRAIN-H-07

受控核查显示，《衍生维度说明》的“表现较好”范围与该题答案纳入的机构范围不一致。该冲突会影响复合经营分析、排名解释和答案比对，由05号在私有异常清单中记录证据和建议，项目负责人确认最终评测处理方式。

公开模板见 [`business_semantics/题库异常清单.csv`](business_semantics/题库异常清单.csv)。

## 六、三项正式业务资产

### 题库表达映射表

至少包含：表达类型、题库原始表达、官方编号、官方标准名称、对应问题编号、审核状态和备注。表达类型包括指标、时间、比较口径、排序或排名表达。

### 衍生指标与复合分析口径表

至少包含：概念名称、概念类型、涉及的基础指标、计算公式、时间范围、机构范围、单位转换、输出形式、来源问题编号和审核状态。

### 题库异常清单

至少包含：问题编号、问题原文、官方答案、异常类型、具体冲突、建议处理方式、最终处理结论和确认人。

当前选择CSV作为人工审核和程序读取的交换格式，原因是结构简单、可在Excel中直接编辑、容易进入版本比较，也能被Python和评测程序读取。公开仓库中的CSV只提供字段结构和脱敏示例；私有完整资产放在：

```text
data/private/business_semantics/
├── 题库表达映射表.csv
├── 衍生指标与复合分析口径表.csv
└── 题库异常清单.csv
```

技术成员读取状态为“已确认”的记录并转换为正式 `metrics.yml`、Schema上下文、Rule、Prompt或评测判分逻辑；其余状态继续保留在候选资产中等待审核。

## 七、协作边界

### 与04号数据库负责人

05号核对官方指标编号、机构编号、日期和实际字段含义；04号负责数据库结构、数据导入、类型、约束、索引和查询性能。05号发现字段或数据不一致时向04号提交问题编号和核查证据。

### 与06号测试负责人

05号提供已确认的业务口径、表达映射和异常信息；06号负责批量评测、答案比对、失败分类、性能和回归报告。测试发现指标、时间或业务计算可能理解错误时，由05号复核口径；SQL、代码、服务或性能问题交给相应技术负责人。

### 与项目负责人和技术成员

项目负责人决定异常题的最终评测策略，并指定技术成员实现配置和确定Rule/LLM路由。05号向技术成员提供已确认业务资产，向06号提供业务口径和异常清单。

## 八、验收标准

Issue #5进入完成状态需要同时满足：

1. 题库中实际出现的非标准表达可以追溯到问题编号和官方定义；
2. 所有收录表达均在题库中具有可核查来源；
3. 衍生指标和复合概念已经明确基础指标、公式、方向、单位、时间和输出形式；
4. 已发现的题目与答案异常全部进入异常清单并具有处理状态；
5. 三项资产结构可被技术成员直接读取和转换；
6. 候选、已确认、需讨论和已驳回状态能够区分；
7. 公开仓库仅包含规则、模板和最小核查示例，完整题库和答案保存在受控目录；
8. 技术实现、批量评测和路由决策均已交接给对应负责人；
9. 04号、06号和项目负责人完成各自边界内的审核；
10. Pull Request合并并向下游成员完成交接。

## 九、首轮交付顺序

1. Codex生成三项资产的候选初稿；
2. 05号先审核高频指标、时间和比较表达；
3. 再审核衍生指标和复合分析概念；
4. 单独复核所有疑似异常题；
5. 与04号核对编号、字段和事实数据；
6. 与06号确认异常题和答案规范化使用方式；
7. 项目负责人确认异常处理结论和技术转换安排；
8. 更新审核状态并提交Pull Request。
