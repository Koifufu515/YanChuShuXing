# 05号负责人：业务语义与题库标准

> 对应 GitHub Issue：#5“官方题库语义映射、衍生口径整理与题库异常审核”
> 资料边界：公开仓库保存规则、模板和最小核查示例；完整题目、官方答案和逐题审核结果保存在 `data/private/business_semantics/`。

## 一、角色定位

05号负责人负责审核官方题库中的自然语言表达、业务指标、时间口径、计算关系和复合分析概念，并将其整理为后端与评测可以引用的标准业务资产。

05号负责人不承担以下工作：

- 不生成、审核或维护 SQL 查询基准；
- 不运行自动化测试、批量评测或性能测试；
- 不决定某道题走 Rule 还是 LLM；
- 不修改数据库表结构、ETL、索引或查询执行器；
- 不重新设计官方指标体系；
- 不擅自修改官方问题或标准答案。

05号的核心价值不是“多写一份说明”，而是把题库语言与官方业务定义之间的关系审核清楚，使技术成员能够据此配置语义层、实现查询，并使测试负责人能够区分技术错误、口径理解错误和官方题目异常。

## 二、工作方式

### Codex 生成候选初稿

Codex读取五张官方表，自动提取：

- 题库中出现的指标别称和省略表达；
- 时间范围、比较基准和排名表达；
- 需要多个基础指标计算的衍生指标；
- 同时涉及多个指标的复合分析概念；
- 题目、官方答案、指标事实和衍生规则之间的疑似冲突。

候选内容必须保留来源问题编号和官方定义引用。Codex只能生成“待审核”初稿，不能直接把候选结果写入正式业务配置。

### 05号逐项审核

05号按顺序对照《机构信息表》《指标清单表》《衍生维度说明》《指标数据表》和《问题答案清单》，确认：

1. 表达是否真的出现在官方题库；
2. 对应的官方指标或衍生口径是否准确；
3. 公式、方向、单位、时间基准和输出形式是否明确；
4. 疑似异常是否有数据证据；
5. 审核状态是待审核、已确认、需讨论还是已驳回。

05号不需要从空白表开始逐条录入，也不需要自己编写提取或转换脚本。技术成员负责把确认后的资产转换为系统配置；测试负责人负责使用异常清单和标准口径开展评测。

## 三、题库表达映射

题库不一定完整使用《指标清单表》和《衍生维度说明》中的标准名称。05号需要记录“题库实际表达—官方指标编号或衍生口径—来源问题编号”的关系。

这不是任意扩写的自然语言同义词库。只有在官方题库中真实出现，并能追溯到具体问题编号的表达才能收录。

### 指标表达示例

`TRAIN-S-07` 使用“存款”这一省略表达，实际查询的是官方指标 `ZB001 各项存款余额`。该映射可以帮助语义层把题库中的短表达连接到权威指标定义，但不能据此自行加入题库中没有出现的“存款规模”“吸储金额”等词语。

### 时间与比较口径示例

`TRAIN-M-01` 使用“和2024年末相比变化了多少”，对应《衍生维度说明》中的“较年初”，其官方定义为“当日值减2024年12月31日值”。审核时需要同时记录比较基准、运算方向和结果单位，不能只写成“同比”或“环比”。

### 排序表达示例

`TRAIN-S-07` 中“三家谁存款最多”表示限定三个机构后，按 `ZB001` 降序取第一。排序表达应记录候选机构范围、排序方向和返回数量，避免把“最多”错误理解成全省排名。

公开模板见 [`business_semantics/题库表达映射表.csv`](business_semantics/题库表达映射表.csv)。

## 四、衍生指标与复合分析口径

此类内容不是简单同义词映射。05号必须明确涉及哪些基础指标、采用什么公式、时间和机构范围是什么、单位如何转换，以及结果应返回单值、列表、排名还是综合分析。

### 新增计算指标示例：存贷比

`TRAIN-M-25` 查询“存贷比”。它不是21项基础指标中的独立字段，应由两个官方指标计算：

```text
存贷比 = ZB002 各项贷款余额 / ZB001 各项存款余额 × 100%
```

该题是指定机构、指定日期的单值查询。审核时需要确认两个基础指标采用同一机构和日期，分母不为零，输入单位一致，输出为百分比，并记录舍入规则由评测规范统一处理。

### 复合业务概念示例：风险指标

`VAL-H-07` 使用“风险指标”这一复合概念，题目要求同时分析：

- `ZB013 不良贷款率`；
- `ZB015 拨备覆盖率`；
- `ZB017 逾期贷款率`；
- `ZB016 资本充足率`。

“风险指标”不是一个可以相加或平均的单值指标。该题应返回四项指标组成的结构化列表，并按题目要求补充排名或全省比较。05号负责固定概念包含关系和业务含义，不负责决定最终 SQL 的 JOIN、聚合或路由实现。

公开模板见 [`business_semantics/衍生指标与复合分析口径表.csv`](business_semantics/衍生指标与复合分析口径表.csv)。

## 五、题库异常审核

05号不能擅自修改官方题目或答案，只负责记录问题编号、原问题、官方答案、冲突点、影响范围和建议处理方式。最终由项目负责人决定：继续按官方答案判分、标记为异常题，或从某项统计中单独排除。

完整原文和答案只保存在私有异常清单。公开仓库使用最小必要摘录和问题编号，既方便核查，也避免公开赛事答案。

### 方向表述冲突：TRAIN-M-46

题目询问“逾期贷款率比不良贷款率高多少”。指标事实显示 `ZB017=1.15%`、`ZB013=1.35%`，逾期贷款率实际更低；官方答案却使用“高出-0.2个百分点”。冲突点是自然语言方向与负数结果不一致，可能影响答案规范化和业务解释。

### 极值判断错误：TRAIN-H-14

题目要求判断2025年成本收入比单日最高和最低。指标事实表中 `25.51%` 是最小值，`37.60%` 是最大值，官方答案将两者方向写反。该题应记录为“极值判断疑似错误”，等待项目负责人确定评测策略。

### 排名口径冲突：TRAIN-H-07

《衍生维度说明》规定“表现较好”为前三名，但该题官方答案把第5名和第7名指标也列入“表现较好”。该冲突会影响复合经营分析、排名解释和答案比对，不能由05号自行改写标准答案。

公开模板见 [`business_semantics/题库异常清单.csv`](business_semantics/题库异常清单.csv)。

## 六、三项正式业务资产

### 题库表达映射表

至少包含：表达类型、题库原始表达、官方编号、官方标准名称、对应问题编号、审核状态和备注。表达类型包括指标、时间、比较口径、排序或排名表达。

### 衍生指标与复合分析口径表

至少包含：概念名称、概念类型、涉及的基础指标、计算公式、时间范围、机构范围、单位转换、输出形式、来源问题编号和审核状态。

### 题库异常清单

至少包含：问题编号、问题原文、官方答案、异常类型、具体冲突、建议处理方式、最终处理结论和确认人。

当前选择CSV作为人工审核和程序读取的交换格式，原因是结构简单、可在Excel中直接编辑、容易进入版本比较，也能被Python和评测程序读取。公开仓库中的CSV只提供字段结构和脱敏示例；私有完整资产放在：

```text
data/private/business_semantics/
├── 题库表达映射表.csv
├── 衍生指标与复合分析口径表.csv
└── 题库异常清单.csv
```

技术成员只读取“已确认”记录转换为系统配置。未经05号确认的候选记录不得进入正式 `metrics.yml`、Schema上下文、Rule、Prompt或评测判分逻辑。

## 七、协作边界

### 与04号数据库负责人

05号核对官方指标编号、机构编号、日期和实际字段含义；04号负责数据库结构、数据导入、类型、约束、索引和查询性能。05号发现字段或数据不一致时提交证据，不直接修改数据库。

### 与06号测试负责人

05号提供已确认的业务口径、表达映射和异常信息；06号负责批量评测、答案比对、失败分类、性能和回归报告。测试发现指标、时间或业务计算可能理解错误时，由05号复核口径；SQL、代码、服务或性能问题交给相应技术负责人。

### 与项目负责人和技术成员

项目负责人决定异常题的最终评测策略，并指定技术成员实现配置和确定Rule/LLM路由。05号只审核业务语义是否忠实于官方资料，不承担技术实现责任，也不向06号提供人工标准查询。

## 八、验收标准

Issue #5不能因为“上传了一份文档”而关闭，必须同时满足：

1. 题库中实际出现的非标准表达可以追溯到问题编号和官方定义；
2. 未加入题库中没有出现的臆造同义词；
3. 衍生指标和复合概念已经明确基础指标、公式、方向、单位、时间和输出形式；
4. 已发现的题目与答案异常全部进入异常清单并具有处理状态；
5. 三项资产结构可被技术成员直接读取和转换；
6. 候选、已确认、需讨论和已驳回状态能够区分；
7. 公开仓库不包含完整官方题库和答案；
8. 文档和Issue中没有要求05号生成或审核SQL、运行测试或决定Rule/LLM路由；
9. 04号、06号和项目负责人完成各自边界内的审核；
10. Pull Request合并并向下游成员完成交接。

## 九、首轮交付顺序

1. Codex生成三项资产的候选初稿；
2. 05号先审核高频指标、时间和比较表达；
3. 再审核衍生指标和复合分析概念；
4. 单独复核所有疑似异常题；
5. 与04号核对编号、字段和事实数据；
6. 与06号确认异常题和答案规范化使用方式；
7. 项目负责人确认异常处理结论和技术转换安排；
8. 更新审核状态并提交Pull Request。
