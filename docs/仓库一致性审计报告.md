## 一、审计结论

当前有效架构是轻量 Ports & Adapters。FastAPI 通过唯一 Composition Root 组装 QueryPipeline，Pipeline 只编排 Context Resolver、SQL Generator、SQL Safety、Database Executor、Result Formatter 和 Audit Logger。

Hybrid 已确认为 Rule First、LLM Extension：标准问题命中 Rule 时不调用 DeepSeek；Rule 未命中后进入两阶段 LLM；LLM 失败返回结构化错误，不再尝试 Rule。Rule 和 LLM 生成的全部 SQL 都经过同一个 SQLGlot Safety Adapter，再进入只读 SQLite Executor。

本轮没有增加业务问题、指标、数据库表或页面功能。修改范围仅包括文档校准、无引用历史文件清理、Safety 内部模型归位和版本标识统一。

## 二、当前有效查询流程

```text
Streamlit
  -> POST /api/v1/query
  -> QueryCommand
  -> YAMLContextResolver
  -> SQLGenerator
       -> Rule
       -> LLM
       -> Hybrid（Rule First）
  -> SQLGlotSafetyChecker
  -> SQLiteExecutor（readonly）
  -> TemplateResultFormatter
  -> QueryOutcome / QueryResponseDTO
```

当前 Metadata 包含配置模式、实际执行器、Rule 命中、路由、失败原因、Provider、模型、耗时和业务语义。旧 `fallback` 字段只为客户端兼容保留，不代表新流程仍执行 LLM 到 Rule 的回退。
