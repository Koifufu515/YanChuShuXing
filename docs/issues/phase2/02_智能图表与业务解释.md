# Issue 草案：智能图表与业务解释

> 状态：候选草案，第一阶段完成且正式评测基线建立前不得创建正式 GitHub Issue。

## 任务目标

根据问题语义、指标口径和查询结果自动选择合适图表，并输出可由数据证据验证的业务结论。图表和解释是结果展示能力，不得复制 SQL 或在前端重新计算指标。

## 依赖条件

- 真实指标、维度、结果字段和主要问题类型已经稳定；
- P1 评测能够识别结果结构与问题类型；
- 产品方案已明确业务结论、关键指标、图表和表格的展示顺序；
- API 可选扩展字段已完成兼容性评审。

## 主要交付物

- 图表推荐规则、图表规格数据结构和安全降级策略；
- 趋势、排名、对比和结构类图表组件；
- 真实指标业务解释模板、口径引用和异常提示规则；
- 数值一致性、空结果、高基数和异常数据测试。

## 涉及模块

Result Formatter、API 可选结果字段、`frontend/`、`docs/product/`、`docs/business/` 和相关测试。Pipeline 只转交结果，不承担图表选择或解释实现。

## 验收指标

图表类型与结果形状一致，超高基数和空结果可安全降级；业务解释中的每个数值能回溯到查询结果，指标口径来自指标字典；主要官方问题类型均有正确示例和自动化测试。

## 负责人角色

产品负责人定义展示规则，业务负责人定义解释口径，前端负责人实现页面，测试负责人核验数值与交互，项目负责人审核公共契约。

## 不能提前开展的条件

真实数据字段、指标口径、结果结构或产品页面层级尚未确定时，不得写死图表和解释；不得让 LLM 在没有结构化证据时自由生成业务原因。
