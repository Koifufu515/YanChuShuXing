# Issue 草案：复杂 NL2SQL 与正式评测

> 状态：候选草案，第一阶段完成前不得创建正式 GitHub Issue。

## 任务目标

基于官方200题建立可重复、可分层、不会泄露答案的评测体系，并用真实基线驱动复杂 NL2SQL 改进。任务重点不是盲目增加 Prompt，而是量化语义、SQL、执行和答案各环节的表现。

## 依赖条件

- #4 已冻结 Real Schema、ETL 和数据版本；
- #5 已完成200题分类、指标口径、Gold SQL 和答案判定；
- #6 已建立 Demo/Real 双环境基线与受控评测位置；
- 项目负责人已冻结评测数据模型和公开边界。

## 主要交付物

- 受控题库加载器、评测执行器和聚合报告；
- 多表、同比环比、排名、区间和组合指标的能力矩阵；
- Rule/LLM 路由、错误分类和逐版本对比；
- 不含官方题目和答案的公开评测说明与脱敏样例。

## 涉及模块

`evaluation/`、`tests/`、`config/`、生成器适配器和评测文档。任何 `application/`、`ports/`、公共 Schema 或指标配置变更必须由项目负责人审核。

## 验收指标

报告至少包含意图识别准确率、Rule 命中率、SQL 生成成功率、SQL 执行成功率、答案正确率、P50/P95 耗时和失败类型，并按题目分区、难度和业务类型分层。首版先记录真实基线，不预设虚假高准确率；测试集答案不得进入 Prompt 或 Rule。

## 负责人角色

业务负责人维护题目、指标和 Gold SQL；测试负责人主导评测实现；数据负责人维护可重复 Real 环境；项目负责人审核核心接口和生成器变更。

## 不能提前开展的条件

Real Schema 未冻结、Gold SQL 未人工对账、官方题库私有加载方式未确认或 Demo/Real 尚未隔离时，不得开始调优和发布准确率结论。
